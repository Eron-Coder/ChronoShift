Problem Statement: Time Switch Escape

Build a simple 2D puzzle-platformer where the player can switch between two versions of the same room: Past and Future.

Some platforms, doors, hazards, or keys exist only in one timeline. The player must switch between timelines to reach the exit.

This is a smaller and more achievable version of the earlier “Time Rift Runner” idea.

Core Requirements

The game must include:

A controllable player character.
At least 3 small levels or rooms.
A Past/Future switch controlled by one key.
At least 3 object types, for example:
Platform
Spike
Key
Door
Moving block
Exit portal
A clear win condition.
A restart button when the player fails.
Minimum Viable Product

A successful MVP can be:

A grid-based or tile-based 2D game.
Simple rectangles instead of art.
Three short puzzle rooms.
Past/Future objects shown using different colors or labels.
